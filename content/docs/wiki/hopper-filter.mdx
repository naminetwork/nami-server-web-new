# 🔧 ホッパーフィルター

<Info>
  このシステムは**国家鯖**と**NamiEarth**で利用できます。
</Info>

ホッパーに名前（フィルター）を設定すると、特定のアイテムだけを通すことができます。
複雑なレッドストーン回路なしで、シンプルかつ高性能な自動仕分け機が作成可能です。

## 1. 設定方法（金床不要）

経験値を消費せず、コマンドも不要な簡単な設定方法です。

1. **スニーク**しながらホッパーを左クリックします。
2. そのまま以下のいずれかの操作を行います。

### アイテムを手に持っている場合
持っているアイテムの名前がフィルターに追加されます。
スニークしたまま別のアイテムを持って繰り返しクリックすることで、複数のアイテムを登録可能です。

### 手に何も持っていない場合
チャット欄が開きます。入力した文字列がそのままフィルターになります。
"null" と入力すると設定したフィルターを削除できます。

## 2. フィルターの書き方

### パターン修飾子

| 記号 | 意味 | 例 |
| :--- | :--- | :--- |
| **なし** | 完全一致 | `stone` → 石だけ |
| **,** | A または B | `stone,dirt` → 石か土 |
| **\*** | 含む | `*wood` → woodを含むアイテム |
| **^** | 始まる | `^oak` → oakで始まるアイテム |
| **$** | 終わる | `$log` → logで終わるアイテム |
| **#** | タグ一致 | `#wool` → 羊毛全般 |
| **~** | ポーション効果 | `~strength_2` → 攻撃上昇2のポーション |
| **+** | エンチャント | `+mending` → 修繕付きアイテム |
| **=** | 表示名 | `=MySword` → 名前を付けた武器など |

### ブール演算子（組み合わせ）

| 記号 | 意味 | 使用例 |
| :--- | :--- | :--- |
| **&** | AND (かつ) | `*oak&!*wood` → oakを含むがwoodは含まない |
| **\|** | OR (または) | `+fortune_3\|+silk_touch` → 幸運かシルクタッチ |
| **!** | NOT (ではない) | `!stone` → 石以外のすべて |

<Tip>
  アイテムの内部名（例: `oak_log`）を確認するには、ゲーム内で **F3 + H** を押してツールチップを表示してください。
</Tip>

---

## 3. 使用例

### 基本的な例
- `*log` : すべての種類の原木
- `enchanted_book&+mending` : 修繕のエンチャント本のみ
- `#villager_plantable_seeds` : 村人が植えられる種すべて

### 高度な例
- **最強のツルハシ選別**:
  `netherite_pickaxe&+efficiency_5&+unbreaking_3&+mending&+fortune_3|+silk_touch`
- **木材製品から原木を除外**:
  `*oak&!*wood` (ボタンやドア、階段のみを通す)

---

## 4. 重要な注意点

### ⚠️ 詰まりの防止
フィルター付きホッパーは**条件に合わないアイテムを一切受け取りません。**
アイテムが溢れて詰まるのを防ぐため、仕分けラインの最後には必ず**「フィルターなしの予備ホッパー（ゴミ箱）」**とチェストを設置してください。

### 🔄 賢い振り分け機能
システムはアイテムを周辺の最適なホッパーへ自動的に振り分けます。
同じアイテムが別のチェストに分散するのを防ぎ、効率的に収納されます。

<Warning>
  OR演算子 `|` は、AND演算子 `&` の内部で優先的に処理されます。複雑すぎる条件はカンマ `,` で条件自体を分けるのがおすすめです。
</Warning>
